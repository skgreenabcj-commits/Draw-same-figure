# 🐱 ずけい れんしゅう — 図形模写学習アプリ
```markdown
小学生受験対策向け「図形模写」練習 Web アプリです。
ドット（グリッド）上の見本図形を見ながら、同じ図形をタッチ/マウスで描くことで「空間認識力・図形再現力」を楽しく鍛えます。
```
---

## ✅ 実装済み機能

| 機能 | 詳細 |
|------|------|
| 4段階レベル | Lv0(4×4・ヒント線2本) / Lv1(4×4・ヒント線2本) / Lv2(4×4・完全回答) / Lv3(5×5・複雑図形) |
| 見本表示 | 左パネルにドットグリッド＋正解線を表示 |
| タッチ/マウス操作 | 点へのスナップ付きドラッグで直線を引く |
| プレビュー線 | ドラッグ中に破線でプレビューを表示 |
| undo / clear | 1本ずつ取り消し・全消し |
| 正誤判定 | 線の向き双方向を考慮した完全一致判定 |
| 正解ポップアップ | gotit.png + 褒め言葉でアニメーション表示 |
| 不正解フィードバック | 赤い正解線をオーバーレイ表示 |
| 5問セッション | ランダム5問を順番に出題、スコア集計 |
| 結果画面 | 5問終了後にスコアとメッセージを表示 |
| Gemini AI問題生成 | APIキー入力でオリジナル問題を生成 |
| Gemini モデル自動フォールバック | 429レート制限時に次モデルへ自動切替・全失敗時は内蔵問題を使用 |
| レスポンシブ対応 | スマートフォン・タブレット・PC 全サイズ対応 |
| Retina対応 | devicePixelRatio を考慮した高解像度描画 |

## 📐 問題フォーマット（座標系）

```json
{
  "level": 1,
  "grid": { "cols": 4, "rows": 4 },
  "lines": [
    { "x1": 0, "y1": 0, "x2": 2, "y2": 0 },
    { "x1": 2, "y1": 0, "x2": 2, "y2": 2 }
  ],
  "hintLines": [
    { "x1": 0, "y1": 0, "x2": 2, "y2": 0 }
  ]
}
```

- 座標は 0始まりの整数インデックス（点のグリッド位置）
- Level0/1/2: cols=rows=4 → x,y ∈ {0,1,2,3}
- Level3: cols=rows=5 → x,y ∈ {0,1,2,3,4}
- hintLines は Level0/Level1 のみ：事前に青で描画される

## 🤖 Gemini AI 問題生成

```text
スタート画面で「AIで問題を作る」をクリック
Gemini API キー（AIza...）を入力して「保存」
キーはブラウザの localStorage に保存（サーバー送信なし）
「はじめる！」でゲーム開始すると自動的に5問を生成
```

### モデルフォールバックチェーンの仕組み

毎回固定モデルを使うのではなく、レート制限（429）発生時に自動で次モデルへ切り替えます。

```text
generateProblems() 呼び出し
      │
      ├─ FALLBACK_MODEL_CHAIN を先頭から順に走査
      │       1. gemini-2.5-flash          （プライマリ）
      │       2. gemini-2.5-flash-lite     （第1フォールバック）
      │       3. gemini-3.1-flash-lite-preview （第2フォールバック・RPD500）
      │
      ├─ 各モデルで最大5回生成試行
      │       └─ 429が3回に達したら即座に次モデルへ移行
      │
      ├─ 401/403: 認証エラー → 即中断して内蔵問題を使用
      │
      └─ 全モデル失敗
              └─ _getFallback() → 内蔵問題バンク（problems.js）を使用
```

### AI生成問題の品質保証

生成された問題は以下のバリデーションを通過したもののみ使用されます。

| チェック項目 | 処理 |
|---|---|
| 端点一致の重複線分 | 除去して線分数が不足する場合は問題を棄却 |
| コリニア重複（同一直線上の部分重複） | 問題全体を棄却して再生成 |
| 交差数制約違反 | 問題全体を棄却して再生成 |
| 線分数不一致 | 問題全体を棄却して再生成 |

## 📋 バージョン更新履歴

**v1.3 — 2026-03-20**
🤖 Gemini AI 問題品質改善（gemini.js v5.3）

グリッドサイズ不一致の修正・コリニア重複対策・プロンプト強化を行いました。

- **グリッド修正（Fix-A）**: `LEVEL_CFG` の `gridN` を修正し、Lv0/1/2 で常に 4×4 グリッドが使用されるよう統一（旧: 5×5 グリッドが生成される不具合を解消）
- **端点重複除去（Fix-B）**: `_normalise` に端点一致による重複線分の除去ロジックを追加。除去後に線分数が不足する場合は問題を棄却して再生成
- **プロンプト強化（Fix-C）**: 端点一致の重複線分禁止ルールを `_buildPrompt` に明示
- **コリニア重複対策（Fix-D）**:
  - `_isCollinearOverlap` / `_hasCollinearOverlap` を新規追加（外積によるコリニア判定 + 1次元区間重複判定）
  - `_normalise` にコリニア重複チェック（Step2）を追加。検出時は問題全体を棄却して再生成を促す
  - `_buildPrompt` にコリニア重複の禁止ルールと具体例（許可・禁止）を追記し、Gemini への指示を強化
- **その他**: `_getFallback` で `PROBLEM_BANK` 側の `grid` 定義を優先するよう修正（防衛的ガード追加）

**v1.1 — 2026-03-20**
🖥️ レスポンシブ対応

スマートフォン・タブレットで表示が切れる・操作できない問題を修正しました。

- `html, body` の `overflow: hidden` を削除し、小画面でもスクロール可能に変更
- `.screen` の `height: 100vh` を `min-height: 100vh` に変更し、コンテンツが溢れても切れないよう修正
- `requestAnimationFrame` を2フレーム待機に変更し、CSS レイアウト計算完了後にキャンバスを描画するよう修正
- `resizeCanvas()` に `offsetWidth` フォールバックを追加し、`display:none` 状態でもサイズが 0 にならないよう対処
- スマートフォン縦持ち（〜600px）では2パネルを縦1列に切り替えるレイアウトを追加
- アクションボタンに `flex: 1 1 auto` を設定し、3ボタンが画面幅に応じて自然に折り返すよう改善
- ResizeObserver を導入し、デバイス回転・ウィンドウリサイズ時にキャンバスを自動再描画するよう変更
- タブレット（601px〜900px）・極小画面（〜360px）向けのブレークポイントを追加

🤖 Gemini モデルの半自動更新

Gemini の無料利用可能モデルが変更された際に、自動で最適なモデルを選択・切り替える仕組みを実装しました。

- `/v1beta/models` API を呼び出し、利用可能なモデル一覧をリアルタイムで取得
- 取得したモデルを優先度リストと突き合わせ、テストリクエストで疎通確認を実施
- 成功したモデル名を localStorage に24時間キャッシュし、毎回テストする無駄を排除
- APIキー保存時にモデルキャッシュを自動リセットし、キー変更時も確実に再選択が走るよう対応
- API エラー発生時にキャッシュを削除し、次回起動時に自動的に再選択するフォールバック処理を追加
- モデル一覧の取得に失敗した場合はハードコードされた優先度リストで動作を継続
