# 🐱 ずけい れんしゅう — 図形模写学習アプリ

小学生受験対策向け「図形模写」練習 Web アプリです。
ドット（グリッド）上の見本図形を見ながら、同じ図形をタッチ/マウスで描くことで「空間認識力・図形再現力」を楽しく鍛えます。

---

## ✅ 実装済み機能

| 機能 | 詳細 |
|------|------|
| 4段階レベル | Lv0(4×4・ヒント線3本) / Lv1(4×4・ヒント線2本) / Lv2(4×4・完全回答) / Lv3(5×5・複雑図形) |
| 見本表示 | 左パネルにドットグリッド＋正解線を表示 |
| タッチ/マウス操作 | 点へのスナップ付きドラッグで直線を引く |
| プレビュー線 | ドラッグ中に破線でプレビューを表示 |
| undo / clear | 1本ずつ取り消し・全消し |
| 正誤判定 | 線の向き双方向を考慮した完全一致判定 |
| 正解ポップアップ | gotit.png + 褒め言葉でアニメーション表示 |
| 不正解フィードバック | 赤い正解線をオーバーレイ表示 |
| 5問セッション | ランダム5問を順番に出題、スコア集計 |
| 結果画面 | 5問終了後にスコアとメッセージを表示 |
| Gemini AI問題生成 | APIキー入力でオリジナル問題を生成 |
| Gemini モデル自動選択 | 利用可能モデルを動的取得・24h キャッシュ・自動フォールバック |
| レスポンシブ対応 | スマートフォン・タブレット・PC 全サイズ対応 |
| Retina対応 | devicePixelRatio を考慮した高解像度描画 |

index.html メイン HTML  css/ style.css スタイルシート（変数・アニメーション含む）  js/ problems.js 内蔵問題バンク (各レベル8問) + ランダム選択  canvas.js Canvas 描画・タッチ/マウス操作  gemini.js Gemini API 連携（問題自動生成・モデル動的選択）  app.js アプリ全体の制御ロジック  images/ cat-mascot.png スタート画面キャラクター  gotit.png 正解時お祝い画像

## 🎮 学習の流れ
```text
スタート画面  
└── レベル選択 (Lv0/Lv1/Lv2/Lv3)  
    └── [任意] Gemini APIキー入力  
        └── 「はじめる！」ボタン  
            └── 問題画面 (5問繰り返し)  
                ├── 見本を見てタッチ/ドラッグで線を引く  
                ├── 「こたえあわせ」ボタン
                ├── 正解 → gotit.png + 褒め言葉 → 次の問題
                └── 不正解 → 緑が正解線 → 次の問題
            └── 5問終了 → 結果画面
```

## 📐 問題フォーマット（座標系）

```json
{
  "level": 1,
  "grid": { "cols": 4, "rows": 4 },
  "lines": [
    { "x1": 0, "y1": 0, "x2": 2, "y2": 0 },
    { "x1": 2, "y1": 0, "x2": 2, "y2": 2 }
  ],
  "hintLines": [
    { "x1": 0, "y1": 0, "x2": 2, "y2": 0 }
  ]
}
座標は 0始まりの整数インデックス（点のグリッド位置）
Level0/1/2: cols=rows=4 → x,y ∈ {0,1,2,3}
Level3: cols=rows=5 → x,y ∈ {0,1,2,3,4}
hintLines は Level1 のみ：事前に薄い青で描画される

```

## 🤖 Gemini AI 問題生成

```json
スタート画面で「AIで問題を作る」をクリック
Gemini API キー（AIza...）を入力して「保存」
キーはブラウザの localStorage に保存（サーバー送信なし）
「はじめる！」でゲーム開始すると自動的に5問を生成
モデル自動選択の仕組み
毎回固定モデルを指定するのではなく、実行時に利用可能な最適モデルを動的に決定します。

APIキー保存 → モデルキャッシュをリセット
      │
      ▼
generateProblems() 呼び出し
      │
      ├─ キャッシュ有効（24時間以内）
      │       └─ キャッシュ済みモデルをそのまま使用
      │
      └─ キャッシュ無効
              │
              ├─ /v1beta/models API でモデル一覧を取得
              │       ↓ 取得失敗時はフォールバックリストへ
              │
              ├─ 優先度リストと突き合わせて候補を生成
              │
              ├─ 候補を上から順にテストリクエスト送信
              │       └─ 成功モデルを localStorage にキャッシュ（24時間）
              │
              └─ generateContent を実行
                      └─ エラー発生時はキャッシュ削除（次回再選択）
フォールバック優先順位:

優先度	モデル名
1	gemini-2.5-flash
2	gemini-2.0-flash
3	gemini-2.0-flash-lite
4	gemini-1.5-flash
5	gemini-1.5-flash-8b
6	gemini-1.5-pro
7	gemini-1.0-pro
🔧 カスタマイズ方法
内蔵問題の追加
js/problems.js の PROBLEM_BANK.level1 / level2 / level3 配列に 上記フォーマットで問題オブジェクトを追加するだけです。

スタイル変更
css/style.css の :root CSS変数を変更することで カラーテーマ・フォント・余白などを一括変更できます。

```

## 📋 バージョン更新履歴


**v1.2 — 2026-03-20**
🎨 UI 改善（回答画面）  
幼児の操作性・視認性向上を目的とした画面改善を行いました。  
  
パネルラベル「見本」を「みほん」にひらがな表記へ変更  
「けす」ボタンを「ぜんぶけす」に変更し、操作内容をより明確に  
「もどす」ボタンを「１つもどす」に変更し、1本単位の取り消しであることを明示  
ヒントメッセージを「のこり２ほんのせんをかこう！」に変更し、全文ひらがな表記に統一  
アクションボタン（「ぜんぶけす」「こたえあわせ」「１つもどす」）を現状比1.5倍のサイズに拡大  
Lv1 のヒント線の色を薄い青（#93BBF5）から見本と同色の濃い青（#1A73E8）に変更し、正解の一部であることを直感的に伝えるよう改善  
  
📐 座標軸ヘッダーの追加（Lv1 / Lv2 のみ）  
幼児が座標の概念を視覚的に学習できるよう、Lv1・Lv2 のグリッドに軸ヘッダーを追加しました。Lv3 は複雑な図形に集中できるよう非表示としています。  
  
横ヘッダー（列方向）：数字（１・２・３・４）を表示  
縦ヘッダー（行方向）：動物アイコン（🐶・🐱・🐰・🐻）を表示  
みほん・かいてみよう の両パネルに同一ヘッダーを表示し、対応関係を把握しやすく設計  
ResizeObserver と連動し、画面サイズ変更時も各セルの高さをキャンバスに自動同期  
  
🔴➡️🟢 不正解フィードバックの改善  
どこが間違っていたかを視覚的に把握しやすくなるよう、フィードバック表示を改善しました。  
正解線の色を赤（#FF2222）から緑（#22BB55）に変更し、「正しい答え」であることを色で区別  
ユーザーの誤答線を (+4px, +4px) オフセットして描画し、正解線と重ならないよう分離することで「どこが違うか」を一目で確認できるよう改善  
フィードバック説明文を「あかい線が　ただしい　せんだよ」から「みどりせんが　ただしい　せんだよ」に変更  

**v1.1 — 2026-03-20**
🖥️ レスポンシブ対応  
スマートフォン・タブレットで表示が切れる・操作できない問題を修正しました。

html, body の overflow: hidden を削除し、小画面でもスクロール可能に変更  
.screen の height: 100vh を min-height: 100vh に変更し、コンテンツが溢れても切れないよう修正  
requestAnimationFrame を2フレーム待機に変更し、aspect-ratio などの CSS レイアウト計算完了後にキャンバスを描画するよう修正  
resizeCanvas() に offsetWidth フォールバックを追加し、display:none 状態でもサイズが 0 にならないよう対処  
スマートフォン縦持ち（〜600px）では2パネルを縦1列に切り替えるレイアウトを追加  
アクションボタンに flex: 1 1 auto を設定し、3ボタンが画面幅に応じて自然に折り返すよう改善  
ResizeObserver を導入し、デバイス回転・ウィンドウリサイズ時にキャンバスを自動再描画するよう変更  
タブレット（601px〜900px）・極小画面（〜360px）向けのブレークポイントを追加  
🤖 Gemini モデルの半自動更新  
Gemini の無料利用可能モデルが変更された際に、自動で最適なモデルを選択・切り替える仕組みを実装しました。  

/v1beta/models API を呼び出し、利用可能なモデル一覧をリアルタイムで取得  
取得したモデルを優先度リストと突き合わせ、テストリクエストで疎通確認を実施  
成功したモデル名を localStorage に24時間キャッシュし、毎回テストする無駄を排除  
APIキー保存時にモデルキャッシュを自動リセットし、キー変更時も確実に再選択が走るよう対応  
API エラー発生時にキャッシュを削除し、次回起動時に自動的に再選択するフォールバック処理を追加  
モデル一覧の取得に失敗した場合はハードコードされた優先度リストで動作を継続
